# Contract: Implementation Brief

## Purpose
Provide engineering-facing implementation direction derived from the UI spec and evidence-backed findings.

## Traceability requirement
Engineering instructions should reference UI requirement IDs or finding IDs whenever a design decision depends on research evidence.

## Required sections
- `objective`: implementation intent and acceptance boundary.
- `deliverables`: required screens/components/artifacts.
- `technical_constraints`: framework, performance, accessibility.
- `component_plan`: decomposition and ownership.
- `instrumentation`: logging/analytics/audit hooks.
- `risks_and_mitigations`: execution risks and controls.
- `handoff_checklist`: items to verify before merge.

## Output quality rules
- Keep requirements measurable.
- Call out assumptions that could invalidate implementation.
- Include explicit non-goals.
