# Contract: Design Research Report

## Purpose
Capture evidence-backed UI research findings in a format that can be traced into design decisions.

## Required sections
- `metadata`: project, screen/scope, author, date.
- `research_goal`: explicit question being investigated.
- `sources`: reference links, screenshots, and provenance notes.
- `observed_patterns`: repeated patterns found across sources.
- `anti_patterns`: patterns to avoid for this context.
- `fit_for_rts`: relevance to context/decision/constraints.
- `open_questions`: unresolved items requiring validation.

## Output quality rules
- Every finding references at least one source.
- Distinguish observation from interpretation.
- Note confidence level for major findings.
