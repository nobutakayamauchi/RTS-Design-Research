# Reference Asset Policy

Reference assets used by RTS Design Research skills must preserve enough provenance for reviewers to reconstruct why a UI recommendation was made.

## Rules
- Record the originating query or prompt for every external UI reference.
- Store source identifiers in generated reports and RTS design decision blocks.
- Use `na` for unavailable URLs or screenshot paths instead of omitting provenance fields.
- Include license, usage, or internal-approval notes before using a reference asset in deliverables.
