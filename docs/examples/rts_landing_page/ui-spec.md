## Section requirements
### Hero
- Headline: "Decision reconstructability infrastructure for engineering teams"
- Subcopy: operational problem statement, no hype claims
- Primary CTA: "View decision block example"

### Problem
- Show decision drift, missing assumptions, audit friction

### Core mechanism
- Present Context → Decision → Constraints → Assumptions → Action → Outcome flow

### Example decision block
- Render a compact YAML excerpt with provenance references

### Ecosystem layers
- RTS core, RTS-Skills, RTS-MCP-Packs, RTS-Hermes-Drive, RTS-Design-Research role

### Use cases
- Incident retrospectives, architecture changes, regulated delivery

### Getting started
- 4-6 practical steps linking to contracts and templates
