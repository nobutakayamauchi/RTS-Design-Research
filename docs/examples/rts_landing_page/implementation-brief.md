# RTS Landing Page Implementation Brief (Example)

## Scope
Single public page describing RTS infrastructure model and artifact outputs.

## Content constraints
- Avoid startup-style superlatives.
- Every claim must map to an artifact or workflow step.
- Keep copy direct and technical.

## UI constraints
- Documentation-adjacent visual style.
- Readable code/artifact snippets.
- Accessible contrast and semantic structure.

## Implementation instructions
- Build page with explicit section anchors matching UI spec.
- Include inline example decision block with source identifiers.
- Link to contracts, policy, and traceability docs.

## Acceptance criteria
- Required sections present and ordered.
- CTA hierarchy: artifact-first, contact-second.
- No unsupported claims.
