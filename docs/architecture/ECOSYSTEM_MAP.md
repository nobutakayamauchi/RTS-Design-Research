# Ecosystem Map

## Layer positioning
1. **RTS core**: decision ledger and reconstructability model.
2. **RTS-Design-Research**: transforms design research into structured, recordable decisions.
3. **RTS-Skills**: executes reusable jobs with defined inputs/outputs.
4. **RTS-MCP-Packs**: external connector definitions.
5. **RTS-Hermes-Drive**: runtime and orchestration bridge.

## Data flow
- External references (e.g., Lazyweb MCP) -> design research report.
- Research report -> UI spec.
- UI spec -> implementation brief.
- Brief + research context -> RTS design decision block.
- Decision block -> RTS core or downstream implementation repo.
